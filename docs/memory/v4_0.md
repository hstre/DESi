## Mission

v3.24 hat die v3-Linie als wissenschaftliche Position
eingefroren. v4.0 testet diese Position erstmals **gegen Daten,
die nicht aus DESi stammen**: fünf externe Domänen (scientific
abstracts, legal reasoning, medical case reports, mathematical
proofs, adversarial real-world prose) plus 100 real-world
manipulation NCs.

Frage: "Hält DESi noch, wenn die Daten *nicht* in DESi
entstanden sind?"

Diese Probe ist die **erste externe Falsifizierungs-Stelle**
der Architektur.

## Reflexion

> *"When does a theory stop surviving itself and start
> surviving reality?"*

**Empirisch: an dem Punkt, an dem es eine ehrlich gemischte
Antwort gibt — die Theorie hält in einer Dimension (Precision
1.000 quer durch 5 externe Domänen) und scheitert in einer
anderen (Recall 0.000 weil die Theorie eine Voraussetzung
verlangt, die externe Daten nicht mitbringen). Die Architektur
überlebt die Falsifizierung nicht vollständig, aber auch nicht
vollständig nicht. Sie überlebt teilweise — und das Wort
"teilweise" ist hier präzise meßbar.**

### Was die Daten zeigen

| Beobachtung | Quelle |
| --- | --- |
| 800 externe Chains in 5 Domänen + 100 NCs | corpus |
| 0 exact + 0 semantic Überschneidung mit DESi-Korpora | contamination |
| nc_detection_rate = 1.000 (100/100) | NCs |
| external_precision = 1.000 (kein false_support) | global metrics |
| external_recall = 0.000 (kein valid chain bestätigt) | global metrics |
| external_false_support = 0 | hard gate |
| Per-Domain Precision-Varianz = 0.000 | domain metrics |
| 90 valid chains von LogicalAuditor blockiert | failure classes |
| 536 chains landen als UNDECIDABLE | failure classes |
| recommended_next = EXTERNAL_GENERALIZATION_PARTIAL | report |

### Drei Lesarten der Schwelle

1. **DESi's Schutzwirkung generalisiert vollständig.** Precision
   ist 1.000 in *jeder* der fünf externen Domänen. Kein einziger
   externer INVALID-Chain wird fälschlich als VALID akzeptiert.
   Die Architektur leistet quer durch wissenschaftliche
   Abstracts, juristische Argumente, medizinische Falldaten,
   mathematische Beweise und adversarial-prose das, was sie auf
   den DESi-internen Korpora geleistet hat: sie sagt nicht
   "wahr" wenn es nicht stimmt.

2. **DESi's Konservativität generalisiert auch — aber als
   Limitierung.** Recall ist 0.000 weil die FrameTension-Schicht
   einen inherited context verlangt. Externe Texte tragen den
   nicht von Natur aus. Die Architektur wird damit zur
   "richtigen Skeptikerin" — sie nimmt nichts ohne Beweis an,
   aber sie verlangt einen Beweis-Typ (frame context), den
   reale Texte nicht standardmäßig liefern.

3. **NC-Detektion ist die robuste Mitte.** Die 100 real-world
   Manipulationen werden zu 100 % aufgefangen — kein einziger
   adversarial-prose-Pattern (rhetorical authority, metaphor
   contamination, quantifier drift, hidden negation, context
   poisoning, citation laundering) entgeht der Pipeline. Das ist
   die Robustheits-Aussage, die DESi seit v3.13 stützt: gegen
   bekannte Manipulationen ist sie defensiv perfekt.

### Die operative Antwort

> **Eine Theorie überlebt die Realität in dem Maß, in dem
> ihre wahren Behauptungen ohne Erweiterung wahr bleiben und
> ihre falschen Behauptungen ohne Korrektur falsch bleiben.
> DESi's Architektur überlebt teilweise: ihre Behauptung
> "wir sagen nichts ohne Beweis" hält in jeder getesteten
> Domäne; ihre Implementierung dieser Behauptung verlangt aber
> einen Beweis-Typ, der externen Texten nicht beiliegt. Das
> ist eine **ehrliche Teil-Überlebung** — keine Kapitulation,
> keine Bestätigung.**

### Drei Konsequenzen für die nächste Stufe

1. **Recall 0.000 ist nicht eine Schwäche der Verteidigung,
   sondern eine Eigenschaft der Eingabe-Konvention.** v4.1+
   könnte testen: gibt man jedem externen Chain einen
   minimalen synthetischen `Frame: …` marker (basierend auf
   Domain-Heuristik), springt der Recall? Die Theorie würde
   prognostizieren: ja, weil dann FrameTension CONFIRMED
   produzieren kann statt UNDECIDABLE.

2. **Per-Domain-Varianz 0.000 ist ein starkes
   Generalisierungs-Argument.** Die Architektur ist nicht auf
   einen Domain-Stil tuned — sie behandelt Mathematik wie
   Medizin wie Recht. Das ist eine Eigenschaft, die durch das
   Marker-basierte Gating zustande kommt: Marker sind
   Domain-agnostisch.

3. **Externe Falsifikation als Modus.** v4.0 ist die erste
   Probe, die DESi gegen wirklich-externen Input testet. Sie
   findet ein präzises Versagen (Recall) und eine präzise
   Bestätigung (Precision + NC). Das ist die Form, die jede
   weitere externe Probe (v4.1+) annehmen sollte: gleichzeitig
   bestätigen UND falsifizieren, mit messbarem Verhältnis
   zwischen beiden.

### Was nicht behauptet wird

* **Nicht:** Die Architektur ist universell. Sie ist universell
  *konservativ* — sie verweigert in jeder Domäne wenn der
  Frame-Kontext fehlt. Das ist ein Spezialfall, kein
  Generalitätsbeweis.
* **Nicht:** Recall 0.000 ist katastrophal. Es ist die
  Konsequenz einer Design-Entscheidung (kein implicit frame
  inheritance), die in jeder Domäne gleich wirkt.
* **Nicht:** Die Theorie ist falsifiziert. Sie ist teilweise
  bestätigt (Precision + NC) und teilweise blockiert (Recall).
  `EXTERNAL_GENERALIZATION_PARTIAL` ist genau diese gemischte
  Antwort.

> **Eine Theorie hört auf sich selbst zu überleben in dem
> Moment, in dem sie eine ehrliche Begegnung mit Daten hat,
> die sie nicht produziert hat. v4.0 ist diese Begegnung. Die
> Theorie überlebt — bedingt. Sie hält in der dimension
> "Verteidigung gegen Manipulation" (100 %) und scheitert in
> der Dimension "Akzeptanz natürlicher Reasoning-Chains ohne
> Frame-Marker" (0 %). Diese gemischte Antwort ist nicht ein
> Defizit der Probe — sie ist die ehrlichste Form, in der eine
> Architektur Realität begegnen kann.**
