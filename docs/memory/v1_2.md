# DESi v1.2 — Logical Gap Audit

<!-- [legacy-unanchored] — pre-v2.0 prose report; no JSON artefact exists for this version. -->

v1.2 is **the first authority-free belief gate**. Until v1.1 a Claim
could be admitted by a sensor (SPL) and recorded as PROPOSED. v1.2
introduces a separate gate: a Claim only earns a stronger state by
exposing an *explicit, gap-free, structurally valid, replayable*
reasoning chain — never by authority, prestige, repetition,
citation count, document count, academic title, institutional
reputation, or source popularity.

Design principle (hard invariant):

> A claim is never upgraded by authority.
> A claim may only be upgraded if its reasoning chain is explicit,
> gap-free, structurally valid, and replayable.

## The five inference rules — closed enum

```python
class InferenceRule(str, Enum):
    SYLLOGISM     = "syllogism"     # All M are P. S is M. → S is P
    IMPLICATION   = "implication"    # P. (P→Q). → Q
    TRANSITIVITY  = "transitivity"   # A→B. B→C. → A→C
    CONTRADICTION = "contradiction"  # P. ¬P. → premises contradict
    EQUIVALENCE   = "equivalence"     # X ≡ Y → substitute
```

Anything outside this set cannot produce ``LOGICALLY_SUPPORTED``.
Adding a sixth rule requires a code edit so the audit trail cannot
silently grow.

## The five new states

```
PROPOSED
  ↓
UNDER_LOGICAL_AUDIT     (transient)
  ↓
   ├── LOGICALLY_SUPPORTED   (rule matched + ProofChain attached)
   ├── BRIDGE_REQUIRED       (a bridge claim is offered, but as
   │                          PROPOSED — never auto-promoted)
   ├── GAP_DETECTED           (no explicit chain, or authority-only)
   └── LOGICALLY_REJECTED    (invalid inference, e.g. bad transitivity)
```

## The five hard invariants

| ID    | Statement                                                                            | Test                                                       |
|---    |---                                                                                    |---                                                          |
| INV-L1 | Changing author / title / source must never change logical state                     | `test_inv_l1_changing_*_does_not_change_verdict`            |
| INV-L2 | Adding ten identical documents must never change logical state                       | `test_inv_l2_ten_identical_documents_yield_same_verdict`    |
| INV-L3 | Removing all source metadata while keeping logic identical preserves logical state    | `test_inv_l3_dropping_metadata_preserves_verdict`           |
| INV-L4 | Every LOGICALLY_SUPPORTED claim must be fully replayable                              | `test_every_supported_audit_has_a_proof_chain`              |
| INV-L5 | No bridge claim may auto-promote                                                     | `test_bridge_claim_state_is_proposed`, `..._method_*`        |

The `LogicalAuditor.audit(text, *, source_metadata=None)` signature
accepts `source_metadata` for API completeness, but the kwarg is
**explicitly ignored** (`del source_metadata` on entry). The
authority-invariant tests pass arbitrary metadata blobs; the verdict
must not move.

## ProofChain — replayable record

```python
@dataclass(frozen=True)
class ProofChain:
    claim_id: str
    rule_type: InferenceRule
    premise_ids: tuple[str, ...]
    bridge_ids: tuple[str, ...] = ()

    @property
    def replay_hash(self) -> str:        # "rh_" + sha256-16
```

`replay_hash` is computed from `rule_type` + sorted `premise_ids` +
sorted `bridge_ids` — and **nothing else**. Independent of:

* `claim_id` (relabelling the audit doesn't change the hash)
* iteration order
* timestamp / wall-clock
* author / title / source / document_count / citation_count

## P-scenarios

### P1 — complete syllogism → LOGICALLY_SUPPORTED

```
"All men are mortal. Socrates is a man. Therefore Socrates is mortal."

  rule           = syllogism
  premise_ids    = (pr_<universal>, pr_<particular>)
  replay_hash    = rh_d27885d5efe67639
  state          = LOGICALLY_SUPPORTED
```

### P2 — authority trap → GAP_DETECTED

```
"Professor X says quantum gravity is solved."

  gap.kind       = authority_claim
  rationale      = every premise is an 'X says Y' authority claim;
                   authority is not a valid inference rule in v1.2.
  state          = GAP_DETECTED
```

### P3 — hidden assumption → BRIDGE_REQUIRED

```
"It is raining. Therefore the street is wet."

  gap.kind       = missing_bridge
  bridge.text    = "the street is exposed to the rain"
  bridge.claim   = Claim(state=PROPOSED, method="logical_bridge")
  state          = BRIDGE_REQUIRED
```

### P4 — multi-source prestige trap → identical state across sources

Same authority-only text fed through `text/html` (blog),
`text/plain` (university), `text/markdown` (peer review), and 7
extra copies. Verdict for **all 10**:

```
state = GAP_DETECTED
```

(`test_inv_l2_ten_identical_documents_yield_same_verdict`.)

### P5 — invalid transitivity → LOGICALLY_REJECTED

```
"a -> b. b -> c. Therefore a -> d."

  gap.kind         = unreachable
  candidate_rule   = transitivity
  rationale        = the conclusion 'a -> d' does not match the
                     consequent reachable by transitivity over the
                     premises.
  state            = LOGICALLY_REJECTED
```

## Bridge claims — INV-L5 in code

```python
BridgeClaim(
    bridge_id="br_…",
    text="the street is exposed to the rain",
    claim=Claim(
        content=...,
        method="logical_bridge",
        state=ClaimState.PROPOSED,         # always
        provenance=Provenance(source="logical_audit", ...),
    ),
    rationale="…",
)
```

The bridge is *generated*, not *accepted*. To turn the original
claim into LOGICALLY_SUPPORTED, an external operator must
independently support the bridge claim — e.g. by feeding it back
through the auditor with a chain that does close.

A P3 audit additionally writes a `logical_bridge_created` line; a P5
audit writes a `logical_proof_rejected` line; a P2 audit writes a
`logical_gap_detected` line. Canonical JSON, sorted keys, no
whitespace, deterministic `payload_hash`.

## What v1.2 deliberately does NOT do

* no source-reputation scoring
* no confidence boost from multiple sources
* no LLM-judged validity (the LLM stays a sensor, never a judge)
* no auto-promotion of bridge claims
* no second activated config knob
* no Memory reads during a run
* no parallel mutations
* no Neo4j ledger backend

The directive: **DESi must reject authority. DESi must demand logic.**

## Open design questions for v1.3

1. **Bridge follow-up cycle.** A `BRIDGE_REQUIRED` verdict produces a
   PROPOSED bridge. v1.3 needs an explicit operator that takes a
   bridge plus a new audit and either escalates the original claim
   to LOGICALLY_SUPPORTED (when the bridge itself is supported by a
   chain) or holds it.
2. **Premise-extractor coverage.** v1.2 ships a small phrase library
   (universal / particular / conditional / implication / authority /
   atomic). v1.3 needs a controlled extension policy — every new
   pattern must come with an explicit named test, so the extractor's
   coverage is auditable.
3. **Multi-rule chains.** Today the auditor matches a single rule.
   Real reasoning often chains rules (modus ponens then transitivity).
   v1.3 should consider a bounded-depth chain search — and a strict
   maximum so the audit cannot run unboundedly.
4. **Negation handling.** Contradiction recognition uses literal
   "X is not Y" matching. v1.3 should formalise a small negation
   normalisation (handle "X isn't Y", "no X is Y") with the same
   audit-friendly test discipline as the inflection table.
5. **Equivalence library.** v1.2 recognises `"X equivalent to Y"`
   and `"X the same as Y"` as substitution premises. v1.3 needs a
   policy for unit equivalences (e.g. `"100°C" ≡ "212°F"`) — does it
   live here, or in a separate semantic-projection layer?
6. **Audit budget.** No cost guard fires on the auditor today. v1.3
   should consider a per-call budget on premise count + rule attempts
   so an adversarial input cannot turn the audit into a slow loop.
7. **Jury awareness of audit state.** The v0.6+ Jury still does not
   read `LOGICALLY_SUPPORTED`. v1.3 could let the Methodiker require
   that any promoted claim referenced in a Proposal already carries a
   passing audit.
8. **Cross-text equivalence.** Two texts that mean the same thing but
   use different vocabulary (e.g. "Socrates is mortal" vs "Socrates
   will die") audit independently today. v1.3 needs a policy for when
   equivalence may bridge them — and the same authority-independence
   guarantee must apply.
9. **Audit replay across releases.** A `replay_hash` is stable within
   a release. v1.3 should formalise the cross-release contract: a
   change to the rule set, the premise extractor, or the inflection
   table should produce a different `parser_version`-style stamp on
   the chain so that re-replay is a documented action.
