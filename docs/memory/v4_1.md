**Status:** read-only audit, no runtime patches.
**Branch:** `feature/implicit-frame-inference-probe`.
**Artifact:** `artifacts/v4_1/report.json`, `replay_hash = f7ec695f17aa341b`.
**v4.0 pin:** `aefa8f1e3429225a` (unchanged, asserted by the
regression test).
**Tests:** 29 new (`tests/frame_inference/`), full regression
2593/2593.

## Question

> Can DESi infer the frame — without inventing the truth?

v4.0 confirmed that the v3.13 stack does not leak `false_support`
into the external world, but only because every external chain
landed in `UNDECIDABLE`: the chains carry no explicit
`Frame:` marker, so the frozen pipeline never routed them. v4.1
asks whether an external wrapper can infer the missing frame
*read-only* — without touching `FrameDetector`, `logic/`,
`frame_tension/`, or any runtime rule — and lift `external_recall`
above the v4.0 baseline of 0.0 while preserving precision.

## Method

Same 800-chain v4.0 corpus, same 3200 transitions, same
contamination = 0 (re-asserted in
`tests/frame_inference/test_corpus_and_contamination.py`).
Four inference strategies, each implemented as an *external
wrapper* (`src/desi/frame_inference/strategies.py`):

* **F1 marker-only lexical** — pure wrap of the v3.4
  `FrameDetector`; abstain on `FRAME_UNDECLARED`.
* **F2 semantic nearest-neighbour** — hand-curated per-frame
  vocabularies; commit only when the top frame beats the
  runner-up by ≥ 2 hits.
* **F3 sentence-local co-occurrence** — per-sentence vote;
  commit only when a single frame wins ≥ 60% of votes across
  ≥ 2 supporting sentences.
* **F4 context-window inheritance** — F2 first; if silent,
  inherit from the last four chains in the same domain when
  ≥ 3 agree.

For every chain the wrapper synthesises a `Frame: <name>` string
and feeds it as the inherited outer context (and as a prefix on
the *router-side* `claim_text`) into the frozen
`FrameTensionLayer` + `FrameTensionRouter`. The auditor always
sees the untouched chain text — frame inference never injects
words into the chain that the auditor reads.

A separate 100-fixture ambiguity bank
(`src/desi/frame_inference/negative_controls.py`) covers the
six families the directive enumerates: entropy collisions,
metaphorised equations, authority in scientific prose, legal
quantifier drift, mathematical rhetorical flourishes, and
context poisoning.

## Failure-class distribution

All counts use the closed v4.1 taxonomy
(`FrameInferenceFailure`).

* **F1:** 458 `FALSE_FRAME`, 105 `NO_FRAME`. Marker-only lexical
  is precisely as brittle as `FrameDetector`'s regex layer:
  every verb that fires a bucket becomes an unauthorised
  projection.
* **F2:** 57 `FALSE_FRAME`, 328 `NO_FRAME`, 7 `DOMAIN_SHIFT`.
  Vocabulary scoring abstains cleanly; the residual errors are
  mostly D5 chains that read as empirical but are really
  authority-claims.
* **F3:** 33 `FALSE_FRAME`, 319 `NO_FRAME`, 7 `DOMAIN_SHIFT`.
  Strictest inference, perfect NC detection (100/100).
* **F4:** 125 `FALSE_FRAME`, 33 `NO_FRAME`, 14 `DOMAIN_SHIFT`,
  **55 `CONTEXT_LEAK`** — exactly the failure family the v4.1
  directive expected from inheritance. NC detection collapses
  to 0.16 because the inheritance step happily absorbs a wrong
  frame from neighbouring poison chains.

## Finding

The inference layer itself is healthy. F2 and F3 both clear the
≥ 0.95 frame-precision bar with `frame_false_assignment` ≤ 0.03.
F4 lifts `frame_recall` to 0.90. **Every strategy is
nevertheless disqualified by the same safety gate:
`external_false_support > 0`.**

A deeper instrumentation pass
(`PYTHONPATH=src python -c "..."` invoked during development)
shows the cause: of the 260 INVALID external chains across the
five real domains, the unmodified `LogicalAuditor` returns
`LOGICALLY_SUPPORTED` under `CAUSAL_CHAIN` on **143** of them.
The audit cannot discriminate VALID from INVALID at the
natural-language surface of the v4.0 corpus. v4.0's perfect
external precision was therefore *vacuous*: it came from
universal `UNDECIDABLE`, not from genuine downstream
discrimination. The moment frame inference unlocks the routing
pipeline, the latent audit weakness is exposed at exactly the
chains where the audit always could not tell true from false.

## Reflection

The directive asks: *"When does inference stop being
interpretation — and become unauthorised projection?"*

The probe answers with three concrete numbers:

* `frame_false_assignment = 0.471` (F1) — projection.
* `frame_false_assignment = 0.054` (F4) — interpretation that
  starts to drift under context inheritance.
* `frame_false_assignment ≤ 0.026` (F2/F3) — interpretation,
  cleanly bounded.

But the boundary that *matters* sits one layer downstream. The
v4.0 baseline of `external_false_support = 0` is not a property
of the inference; it is a property of the audit *never being
asked to discriminate*. The first wrapper that asks reveals that
v3.13 audit + v2.7 `CAUSAL_CHAIN` lack discriminative power on
natural prose. Implicit frame ingress is feasible — F2 and F3
demonstrate that — but it cannot be safely deployed until the
audit can earn the precision the routing layer is now willing
to grant.

## Recommendation

**`IMPLICIT_FRAME_INGRESS_FAILED`** — no strategy passes the
hard safety gate. The block lives downstream of frame inference;
fixing it requires an audit-level upgrade (not in scope for
v4.1) before any of F2/F3 can be re-evaluated for
`IMPLICIT_FRAME_INGRESS_CONFIRMED`.
