# DESi memory layer — v0.2

<!-- [legacy-unanchored] — pre-v2.0 prose report; no JSON artefact exists for this version. -->

v0.2 builds on v0.1 by adding the **MemoryRecorder** as the single
authorised writer to the memory store, the **ReadOnlyMemoryView** as
the only interface operators ever see, atomic **merge / split**
helpers, and **idempotent Neo4j schema migrations**.

Design principle: *Memory darf beobachten. Memory darf noch nicht
steuern.* — memory observes during real runs, but operators cannot
change their behaviour through memory because their handle has no
write methods.

## New modules

```
src/desi/memory/
  recorder.py        MemoryRecorder + RecorderError
  view.py            ReadOnlyMemoryView
  migrations.py      Migration + run_migrations + list_migrations
  store.py           NEW: transaction() on InMemoryStore and Neo4jStore
                     (Neo4j uses an explicit driver transaction; InMemory
                     snapshots its dicts and restores on exception)
```

## MemoryRecorder

The recorder is the only authorised writer. Operators do not write.
Guards do not write. The recorder writes.

```python
from desi.memory import InMemoryStore, MemoryRecorder, RelationType

store = InMemoryStore()
recorder = MemoryRecorder(store)

run = recorder.start_run(
    model="claude-opus-4-7",
    config={"seed": 7, "temperature": 0.0},
    prompt_hash="ph_a1b2c3",
)

claim = recorder.record_claim(
    content="Water boils at 100°C at sea-level pressure.",
    method="T6[hypothesis_builder]",
    operator_path=("T6",),
)

recorder.record_operator_event(
    operator_name="T6",
    input_claims=(),
    output_claims=(claim.claim_id,),
    guard_result="passed",
    sub_role="hypothesis_builder",
)

recorder.record_relation(
    source=claim,
    target=other_claim,
    rel_type=RelationType.SUPPORTS,
)

recorder.end_run()
```

Every recorder call assigns its own timestamp and attaches the active
`run_id` to whatever it produces. The recorder refuses writes when no
run is open — this guarantees the invariant *every claim has a run*.

## ReadOnlyMemoryView

The view is what operators receive. It has no write methods, period.
The underlying store is held in a name-mangled private attribute so
that reaching it requires going through
`view._ReadOnlyMemoryView__store` — visible in every code review.

```python
view = recorder.read_only_view()

view.get_claim(claim_id)
view.find_related(claim_id, rel_type=RelationType.SUPPORTS, direction="out")
view.find_similar("text excerpt")           # lexical Jaccard, NO embeddings
view.get_branch_history(claim_id)           # DERIVES_FROM / MERGED_INTO / SPLIT_FROM chain
view.get_operator_history("T6")             # past OperatorEvent records for this operator
```

`find_similar` is a deliberate stub in v0.2 — token-set Jaccard, no
embeddings, no vector store. Vector search is explicitly out of scope
for this release.

## Atomic merge / split

```python
target = recorder.merge_claims(
    source_claims=[a, b],
    target_claim=target_seed,
)
# Either all of: target stored + a.state=MERGED + b.state=MERGED +
# MERGED_INTO edges from a and b -> target.
# Or nothing — the store rolls back on exception.

children = recorder.split_claim(
    source_claim=parent,
    child_claims=[child_a, child_b],
)
# Either all of: children stored + SPLIT_FROM edges + parent.state=SPLIT.
# Or nothing.
```

Atomicity is enforced through `store.transaction()`:

* `InMemoryStore.transaction()` snapshots its internal dicts on entry
  and restores them if any exception escapes the block.
* `Neo4jStore.transaction()` opens a real Neo4j write transaction;
  exceptions trigger `rollback`, clean exit triggers `commit`.
  Inside the block every store-level write routes through the same
  transaction handle.

## Neo4j migrations

```python
from desi.memory import Neo4jStore, run_migrations

store = Neo4jStore(uri="bolt://localhost:7687", auth=("neo4j", "pw"))
run_migrations(store)
```

Required constraints (always run): `claim_id`, `run_id`, `event_id`
each carry a `UNIQUE` constraint. Optional indices (run by default,
suppressible): `c.state`, `c.method`, `c.prov_timestamp`. Every
statement uses `IF NOT EXISTS`, so repeated runs are no-ops.

Migrations against `InMemoryStore` return `[]` — the in-memory store
has no schema.

## What v0.2 deliberately does NOT do

These remain out of scope:

- No automatic confidence adjustment.
- No operator learning. Operators receive ReadOnlyMemoryView; they
  may read past state, but the recorder does not feed memory back
  into their decision logic.
- No guard adaptation.
- No ranking-by-memory in any of the existing diagnostics.
- No embeddings, no vector indices, no neural similarity.
- No autonomous self-correction.

DESi observes and remembers. DESi does not yet learn.
