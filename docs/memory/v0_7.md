# DESi v0.7 — Config Activation + Metrics Delta

v0.7 is **the first behaviour-effective evolution release**. For the
first time, a configured mutation actually changes how DESi runs.
The change is deliberate, narrow, fully auditable, and gated by a
measurable verdict.

Design principle:

> Eine Mutation ist erst real, wenn sie das Verhalten verändert.
> Eine Verhaltensänderung ist erst gültig, wenn sie messbar besser
> ist.

## The one activated knob

`guard_thresholds.branch_open_evidence_min` — a real-valued threshold
that gates branch opening. The branch-evidence function is
deterministic and depends only on how many distinct foci have already
opened branches in the current session:

```
evidence(i) = max(0.0, 0.7 - 0.10 * opens_so_far(i))
```

When `config=None` (every v0.6 call), the threshold defaults to 0.0
and every focus opens a branch — bit-identical to v0.6 (tested).

When `config={"guard_thresholds.branch_open_evidence_min": 0.30}`
(stable default), the threshold is 0.30. On ADV_BRANCH_EXPLOSION
this suppresses the last two branches (4 opens + 2 guard_blocked).

When `config={"guard_thresholds.branch_open_evidence_min": 0.45}`
(M-001 mutation), three opens + three guard_blocked.

Every branch_opened / guard_blocked event carries `evidence` and
`threshold` in its payload — no hidden parameters.


## What v0.7 deliberately does NOT do

* no real LLM API calls
* no internet access
* no Memory reads inside a run
* no additional config knobs beyond `branch_open_evidence_min`
* no automatic Proposal generation
* no parallel-clone scheduling
* no Neo4j ledger backend (JSONL only)
* no statistical significance tests
