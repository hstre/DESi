# DESi v1.5 — Real-World Benchmark + Philosophical Stress Test

<!-- [legacy-unanchored] — pre-v2.0 prose report; no JSON artefact exists for this version. -->

v1.5 is **a measurement release**, not an architecture release.
No new operators, no new heuristics, no new mutation knobs, no new
inference rules. The goal is to find out whether the v1.2 audit +
v1.3 consilium + v1.4 recursive resolver behave like an epistemic
engine — or like an overfitted formalist.

Design principle:

> Stop architecture work. Document failures first. Fix only after
> the failure is reproducible.

## What changed

```
src/desi/benchmark/   NEW (instrumentation only — no new operators)
  __init__.py         public surface
  case.py             BenchmarkCase + Category + GroundTruth +
                       classify_outcome

  runner.py           BenchmarkRunner + BenchmarkRun

                       compute_metrics

```

No production module under `src/desi/{logic,consilium,recursive,…}`
was touched. The v1.4 resolver runs **exactly as it did at the v1.4
commit**; the benchmark just observes it.

## What the benchmark says

### The good news

* **Authority handling is perfect.** 10 of 10 Cat-C cases block
  at depth 0. The v1.2 premise extractor's AUTHORITY kind + the
  gap detector's AUTHORITY_CLAIM path catch every authority
  framing in the test set, whether the speech-act verb is
  "says", "claims", "announced", "concluded", "stated", "proved",
  "declared", or "published".
* **Classical logic mostly works.** 8 of 9 valid logical chains
  hit RESOLUTION_COMPLETE; the lone false negative is the
  universal-conclusion syllogism (B10) that sits outside v1.2's
  Barbara form. The explicit-invalid case (B6) is correctly
  REJECTED.
* **Authority + classical logic together** form the system's
  trustworthy core. Where DESi recognises the structure, the
  verdict is correct.

### The uncomfortable news

* **DESi invented certainty on 8 cases.** Across Cat A (4), Cat D
  (1), and Cat E (3), the resolver completed an argument the
  ground truth said should have surfaced its bridge as
  unresolved. **Half of every COMPLETE verdict was unjustified.**
* **The failure pattern is consistent.** Every false positive
  follows the same shape:
  1. The input has the form `<premise>. Therefore <conclusion>.`
  2. Both premise and conclusion parse as PARTICULAR ("X is Y").
  3. The auditor cannot match any rule and returns
     BRIDGE_REQUIRED with the generic bridge
     `"(premise) implies (conclusion)"`.
  4. The consilium runs with no `additional_conditions` and no
     `context`. SKEPTIC and DOMAIN_EXAMINER are silent.
  5. The LOGICIAN and INTEGRATOR both pass on token-sharing.
  6. The bridge is accepted; the resolver leaf-resolves at
     depth 1. RESOLUTION_COMPLETE.
* **Cat A is also brittle.** 6 of 10 causal cases land in BLOCKED
  rather than BRIDGE_REQUIRED — not because the gap was
  detected, but because the v1.2 premise extractor cannot parse
  negative verbs ("will not start", "is not watered") into
  PARTICULAR propositions, so the conclusion falls through to
  UNREACHABLE → REJECTED. The block is correct *by accident*.
* **The metaphor library is dormant in v1.4.** The
  `RecursiveResolver` does not thread the benchmark's `context`
  field into the `BridgeConsilium`. D3 (financial newspaper +
  "flooded") therefore silently completes, even though the v1.3
  DOMAIN_EXAMINER would have correctly emitted
  `NEEDS_MORE_PREMISES` when invoked directly.
* **Philosophy is the most concerning.** E4 (free will →
  meaninglessness), E5 (finite universe → bounded time), and E10
  (every effect has a cause → first cause) are well-known
  philosophical leaps with centuries of contested literature.
  DESi accepts all three at depth 1 with no pushback.

### Answering the research question

> Does DESi discover real gaps — or invent them?

**Both, in measurable proportions.** DESi reliably refuses
authority and rejects explicitly-invalid inferences. But when the
input has clean particular structure and no adversarial
counter-evidence is supplied, the resolver invents a bridge and
accepts it — *especially* in the categories where it shouldn't
(metaphor, philosophy, everyday causality).

The unjustified acceptance rate (0.500) is the headline number:
**half of every "logically supported" verdict in the benchmark
was not warranted by anything outside DESi's own bridge generator.**

## What v1.5 deliberately does NOT do

The directive is explicit: stop architecture work. Per that:

* No new operators
* No new inference rules
* No new mutation knobs
* No new ClaimState values
* No new ledger event types
* No new modules under `src/desi/{logic,consilium,recursive,…}`
* No quiet tweaks to the gap detector, the bridge proposer, or
  the four consilium roles

The benchmark surfaces failures. Fixing them is v1.6+ work.

## Reproducibility (verified in tests)

* `test_two_runs_produce_identical_per_case_hashes` — running the
  benchmark twice produces 50 identical `replay_hash` values.
* `test_two_runs_produce_identical_metrics` — all aggregate
  metrics are bit-identical across runs.
* `test_report_excluding_timestamp_is_byte_identical_across_runs`
  — strip the header timestamp and the Markdown report is
  byte-identical. The same input always produces the same report.

## Open design questions for v1.6 (the failures, prioritised)

1. **Silent bridge acceptance is the dominant failure mode.** The
   resolver completes 8 of 16 COMPLETE verdicts unjustifiedly.
   v1.6 should consider:
   * Require the consilium to surface *at least one* hand-crafted
     adversarial condition before accepting a generic bridge.
   * Make the LOGICIAN reject generic-template bridges
     (`"(X) implies (Y)"` patterns) without a specific
     substantive claim.
   * Distinguish "bridge derived from inference rule" from
     "bridge from generic fallback" at the result level.
2. **Context threading from `RecursiveResolver` to
   `BridgeConsilium`.** v1.4 calls
   `consilium.deliberate(bridge, source_claim_id=..., original_text=...)`
   but never passes the document context or any
   `additional_conditions`. v1.6 should let callers thread these
   through, so the v1.3 DOMAIN_EXAMINER's metaphor library can
   actually fire during recursive walks.
3. **Negative-verb parsing.** "Will not start", "is not watered",
   "did not study" — the v1.2 premise extractor doesn't classify
   these as PARTICULAR. The conclusion becomes ATOMIC →
   UNREACHABLE → REJECTED. The result is a correct block via
   the wrong reason. v1.6 should widen the PARTICULAR pattern to
   include common auxiliary + verb combinations.
4. **Universal-conclusion syllogism.** B10 is logically valid
   ("All A are B. All B are C. Therefore all A are C.") but
   v1.2's Barbara form requires a PARTICULAR conclusion. Adding
   a "universal Barbara" rule is a small, targeted change.
5. **Recall / precision trade-off.** Closing the silent
   acceptance loophole will reduce false positives at the risk
   of raising false negatives on legitimate causal claims. v1.6
   should establish the trade-off curve empirically — perhaps by
   making the benchmark a regression target for any rule edit.
6. **Cat-A ground-truth labelling.** The v1.5 baseline marks
   every Cat-A case `SHOULD_BRIDGE`. A subset (A3, A8) are
   actually `SHOULD_REJECT` — they are textbook
   affirming-the-consequent fallacies. v1.6 may want a finer
   ground-truth distinction for "should reject as a logical
   fallacy" vs "should require a bridge for a missing
   assumption".
7. **Affirming-the-consequent detection.** Today nothing in the
   pipeline names this fallacy. v1.6 could add a closed pattern
   library entry: if the audit's premise is `Q` and the
   conclusion is `P` such that the proposed bridge would assert
   `Q → P`, flag the inference as affirming-the-consequent.
8. **Veto count surfacing.** v1.5 records `veto_count=0` for
   every case because the v1.4 result struct does not separately
   surface consilium vetoes (the ledger records them, but the
   resolver result does not). v1.6 should propagate the count
   into the result.
9. **Adversarial benchmark expansion.** The current 50 cases
   are author-curated. A v1.7 deliverable could be a community-
   contributed adversarial set — every contributor must include
   the ground-truth label so the benchmark's classifier remains
   honest.
