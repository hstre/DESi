## Anker-Schema (Aufgabe 1)

```
[claim_anchor: artifact=<path>, field=<json-path>, expected=<value>]
```

(In real anchors the prefix is `[claim-anchor:` with a hyphen — the
example above uses an underscore so the parser does not treat the
schema documentation itself as a claim.)

Eigenschaften:
- **single-line** (anchor + claim auf derselben Zeile, damit v3.0
  self_audit den Artefakt-Hint per `_line_artifact_hint` aufgreift)
- **whitespace-insensitive** zwischen den Kommas
- **case-sensitive** auf den Schlüsseln (`artifact`, `field`,
  `expected`)
- Felder unterstützen Dot-Pfad, `len:KEY`, und `[N]`-Indices

Legacy-Marker (Aufgabe 3): `[legacy_unanchored]` als HTML-Kommentar
unter dem H1-Heading jedes pre-v2.0-Dokuments.

## Aufgabe 9 — Reflexion

> *"When does documentation stop being narrative and become evidence?"*

**Dokumentation wird Evidenz, wenn jede Behauptung einen
maschinell verifizierbaren Anker trägt, der auf eine konkrete
Artefakt-Quelle zeigt.**

Drei operative Schwellen lassen sich aus der v3.0 → v3.1
Beweislage ableiten:

1. **`self_deception_rate < 0.20`** — eine **objektiv messbare
   Untergrenze**, oberhalb derer das Dokumentations-System mehr
   unverifizierbare als verifizierbare Behauptungen produziert.
   v3.0 lag bei 0.314 (31.4 % unverifizierbar). v3.1 liegt bei
   0.051 (5.1 %).
2. **`verified_anchors / total_anchors == 1.000`** — **kein
   einziger** Anker darf auf ein fehlendes Artefakt, einen
   fehlenden Feldpfad oder einen falschen Wert zeigen. v3.1: 36/36.
3. **`drift_findings == 0`** — **keine** versionübergreifende
   Behauptung darf sich still ändern. v3.1: 0 Drifts.

Wenn diese drei Bedingungen gleichzeitig gelten, dann ist die
Dokumentation eines Repositories nicht mehr ein narratives
Begleitwerk, sondern ein **prüfbares Bezugssystem**: jede Zahl im
Text führt rückwärts zu einer Datei mit demselben Wert; jede Datei
führt vorwärts zu mindestens einer Doku-Stelle, die sie
referenziert; und die Konsistenz beider Richtungen ist maschinell
testbar.

**Vor v3.1:** "DESi behauptet X." war ein Akt des Vertrauens.
Ein menschlicher Reviewer musste prüfen, ob X stimmt.

**Nach v3.1:** "DESi behauptet X" trägt einen Anker
`[claim_anchor: artifact=A, field=F, expected=X]`. Das ist eine
operationale Aussage: "Wenn du `A.F` ausließt und das nicht
`X` ist, dann ist die Behauptung falsch — und das ist
maschinell entscheidbar." Vertrauen wird durch Verifizierbarkeit
ersetzt.

Die 5.1 %-Restquote sind ehrliche Lücken: 19 v0.x/v1.x-Dokumente
sind explizit als `[legacy_unanchored]` markiert (kein JSON-Artefakt
hat je existiert) und 23 weitere Claims im v2.x/v3.x-Korpus sind
narrative Hilfssätze ohne fassbares Anker-Ziel. Beides ist sichtbar,
benannt, und nicht stillschweigend.

> **Documentation stops being narrative the moment "we said X"
> becomes "X is at A.F".**
