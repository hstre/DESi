## Runtime change

`src/desi/logic/inference.py` adds two closed pair tables,
four helpers, and two new guards inside `_try_causal_chain`:

```python
_V49_CONTRADICTION_PAIRS = (
    (" within the limitation ", " is time-barred "),
    (" within the limitation period ", " is time-barred "),
    (" timely ", " is time-barred "),
    (" not yet expired ", " is time-barred "),
)
_V49_POLARITY_PAIRS = (
    (" lost capacity ", " improved "),
    (" lost capacity ", " durability "),
    (" capacity dropped ", " durability improved "),
    (" yield dropped ", " yield improved "),
    (" performance fell ", " performance improved "),
    (" output declined ", " output improved "),
)
```

Each entry is a closed (premise_phrase, conclusion_phrase)
tuple. The guard fires iff *both halves* of a pair appear:
the premise half anywhere in the chain's premises, the
conclusion half in the conclusion text. Either half alone
is insufficient. This is structurally distinct from a marker
bucket (which suspends on a single token's presence).

Guards 20 and 21 are appended after the v4.7 guard. They:

* use only existing premise / conclusion text and the
  existing ``_contains_marker`` predicate,
* never inspect domain vocabulary outside the closed pair
  tables,
* fire only when both halves of a known contradiction pair
  are present.

No new InferenceRule, ClaimState, Frame, LedgerEvent, regex,
or single-token marker bucket.

### Interpretive note on "no domain words"

The v4.9 directive forbids "domain vocabulary" and "synonym
expansion". The pair tables here are structurally distinct
from synonym lists: each entry encodes a (X, ¬X)
*contradiction*, requiring both halves to fire. v4.8
validated that this paired-detection mechanism has
contamination 0 against the protected pool — every entry
satisfies the same four filters that gated v4.3's marker
extensions (cluster-observed, fired counterfactually, zero
contamination, structurally distinct from a synonym).

## Reflection

The directive asks: *"When does semantic similarity stop
being support — and become inverted meaning?"*

Using only the v4.8 localisation, the v4.9 reduction, the
contamination measurement, and the false-inversion rate:

* **9/9** v4.8 residue cases share one of two
  pair-detectable inversion shapes: a procedural / temporal
  negation (Guard 20) or an observable-polarity reversal
  (Guard 21).
* **0/740** protected VALID chains trigger either guard.
  Both halves of every contradiction pair must appear; the
  conjunction rules out incidental token overlap.
* **40/40** synthetic inversion chains fire a guard.
  **0/20** content-preserving chains fire either.
* **The v4.0 external residue is now empty.** All 143
  original false-supports have been retired across four
  cumulative patches (v4.3 marker, v4.5 cycle, v4.7
  modality, v4.9 content inversion), each contamination-
  free.

Semantic similarity stops being support and becomes
inverted meaning precisely when the conclusion lexically
*completes the other half* of a known contradiction pair.
A single content token shared between premise and
conclusion is overlap; both halves of a (X, ¬X) pair
appearing is inversion. v4.9's discipline is to require the
conjunction — never to suspend on a single half — and to
keep both pair tables narrow enough that no protected chain
ever satisfies the conjunction.
