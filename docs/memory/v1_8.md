# DESi v1.8 — Authority Speech Acts + Parser Reality Check

<!-- [legacy-unanchored] — pre-v2.0 prose report; no JSON artefact exists for this version. -->

v1.8 is **a comprehension fix**. v1.7 surfaced the v1.7 reflection
finding: 8 of 10 Cat-C authority traps blocked correctly but for
the wrong reason — the parser only recognised the literal verb
"says" as an authority speech-act. v1.8 closes that gap with a
closed lemma library and short-circuits authority premises to a
principled `LOGICALLY_REJECTED` verdict before any inference,
bridge generation, or consilium call.

Design principle:

> DESi muss unterscheiden zwischen
> (1) echter Authority-Erkennung
> (2) zufälligem BLOCK durch Parser-Unverständnis.

correctly, but it has classified them as "factual claims" not
"authority claims".

## Aufgabe 2 — Closed speech-act library

```python
AUTHORITY_SPEECH_ACT_LEMMAS: frozenset[str] = frozenset({
    "says", "said",
    "states", "stated",
    "claims", "claimed",
    "declares", "declared",
    "concludes", "concluded",
    "announces", "announced",
    "publishes", "published",
    "proves", "proved",
    "reports", "reported",
})
```

Plus the special preposition form `According to X, Y` / `According
to X: Y`.

The set is closed. Adding a lemma requires a code edit, which is
itself an audit event. The pinned test
`test_authority_speech_act_lemmas_is_closed` asserts the exact
size (18).

## Aufgabe 3 — Parser integration

`desi.logic.premises._classify()` gained two new regex tests
**before** the v1.7 negation patterns (so authority preambles
like "Wikipedia states that Pluto is not a planet" are no longer
greedy-captured into PARTICULAR(`pred="not a planet"`)):

```python
_RE_AUTHORITY_ACCORDING_TO  # "According to X, Y"
_RE_AUTHORITY_SPEECH_ACT    # "<speaker> <lemma> [that] <claim>"
```

Both produce `PremiseKind.AUTHORITY` with the speaker preserved
verbatim and the claim text canonicalised into the predicate.

Detection is **speech-act based**. The directive explicitly forbade
special-casing speaker names; my implementation does not look at
"Professor", "Nature", "Nobel", or any speaker token.

## Aufgabe 4 — Audit integration

`LogicalAuditor.audit()` now short-circuits authority premises
**before** any rule-set attempt:

```python
propositions = self._extractor.extract(text)
if any(p.kind is PremiseKind.AUTHORITY for p in propositions.premises):
    return self._reject_authority(ac_id, text, propositions)
```

`_reject_authority` returns an `AuditResult` with:

* `state = LogicalState.LOGICALLY_REJECTED`
* `gap = Gap(kind=AUTHORITY_CLAIM, rationale=…)`
* ledger event: `LOGICAL_PROOF_REJECTED` with `reason="authority_claim"`

The resolver's `_classify_audit_block` was extended so that
`LOGICALLY_REJECTED + AUTHORITY_CLAIM gap` → `BlockingReason.AUTHORITY_CLAIM`.

This means: authority **never reaches**
* the inference rule set (try_each_rule is bypassed)
* the gap_detector's bridge proposer
* the consilium (no bridges = no `consilium.deliberate()` call)
* the recursive resolver's BRIDGE_REQUIRED path

The early exit is the earliest possible audit-time veto.

## Reflection question

> *"If a system rejects authority without recognizing authority,
>  has it rejected authority — or merely failed to understand
>  language?"*

v1.7 answered: **it failed to understand language.** 8 of 10 Cat-C
cases blocked correctly while their predicates were misclassified
as factual particulars.

v1.8 answers: **it now recognises authority — within a closed,
inspectable speech-act library of 18 lemmas plus the "according
to" form.** Outside that library (e.g., "X opined that Y", "X
suggested that Y"), DESi will again block "for the wrong reason".
The library is intentionally narrow so the gap stays visible: any
future authority verb the directive endorses must be added
explicitly, and `test_authority_speech_act_lemmas_is_closed`
pins the size so silent growth isn't possible.

The deeper answer: DESi has moved from "lucky" to "honest". It
blocks Cat-C for the right reason on the corpus we can name. It
will still fail on speakers we haven't anticipated. The audit
trail now distinguishes the two — that is the entirety of v1.8's
deliverable.

## Open design questions for v1.9

1. **Reporting verbs outside the library.** "opined", "suggested",
   "argued", "speculated", "hypothesised", "asserted", "maintained",
   "noted" — all plausible authority cues that v1.8 does not yet
   recognise. v1.9 should either extend the library (with the same
   audit-trail discipline) or formalise a denotation for "verbs we
   don't recognise → PARSER_UNSUPPORTED_FORM, no fake AUTHORITY
   classification".
2. **Multi-clause authority.** "Smith said X. Therefore X." is
   handled; "Smith said X and Jones agreed. Therefore X." is not.
   The extractor only sees the first sentence as authority and
   then a particular conjunction. v1.9 may want explicit support
   for conjunctive authority chains.
3. **Negated authority.** "Smith did not say X" — v1.7's negation
   patterns will capture this as `PARTICULAR(smith, "not say X")`,
   not as a negated authority claim. v1.9 should decide whether
   "did not say" should classify as anything authoritative at all
   (probably no — non-authority).
4. **Quoted authority.** `Smith: "X is true."` — direct-quote
   form. v1.8 doesn't handle it.
5. **`AUTHORITY_PREMISE` confidence.** The v1.2 Premise model has
   a `predicate` field but no separate `embedded_claim`. v1.9
   could add an `embedded_claim` accessor so downstream consumers
   (e.g., a future Jury that quarantines but does not promote)
   can read the authority's actual claim cleanly.
6. **Multi-language authority.** Currently English-only. v1.9
   could extend the library to German equivalents (`sagt`, `behauptet`,
   `erklärte`, `verkündete`) under the same closed-list discipline.
7. **Authority chain with non-authority co-premise.** "Smith said
   X. Y is a Z. Therefore Z is X." Today the audit blocks any
   AUTHORITY-bearing input. A future release might allow the
   non-authority premises to derive an unrelated conclusion while
   still rejecting any conclusion that *depends on* the authority.
8. **Inflection table interaction.** The v1.6 inflection table
   normalises plurals (philosophers → philosopher). Could
   accidentally interact with authority verbs ("publishes" →
   "publishe"). v1.9 should add a denylist for verbs to prevent
   the stem-strip from silently corrupting the library.
9. **`AUTHORITY_PREMISE` ledger payload.** v1.8 logs
   `LOGICAL_PROOF_REJECTED { reason: "authority_claim" }` but
   doesn't surface the recognised speaker(s) in the payload. v1.9
   could include the speakers list so downstream audit tools can
   classify by source.
