### NaturalnessVector (Aufgabe 1)

Acht Features pro Chain:

1. `marker_density` — marker tokens / total content tokens
2. `marker_entropy` — Shannon entropy across marker classes
3. `lexical_redundancy` — repeated content tokens / total
4. `synonym_stacking` — adjacent sentences sharing marker class
5. `transition_variance` — variance of sentence-pair lexical
   distances
6. `frame_switch_frequency` — link-type switches / link count
7. `unknown_link_ratio` — UNKNOWN links / total links
8. `explicitness_score` — weighted sum of markers

## Reflexion

> *"When does explanation become performance rather than
> reasoning?"*

**Empirisch: Erklärung wird zur Performance in dem Moment, in
dem die Marker-Dichte über die Schwelle steigt, die natürliche
kausale Sprache verlässt — und sie bleibt Reasoning so lange,
wie die Marker-Dichte unterhalb der natürlichen Sparsity
bleibt. Die v3.18-Daten zeigen das numerisch: valid_but_sparse
Chains (100 % akzeptiert) sind natural; over_explained_but_true
Chains (100 % flagged) sind Performance — selbst wenn ihre
Conclusion stimmt.**

### Was die Daten zeigen

| Beobachtung | Quelle |
| --- | --- |
| 480 chains, 1018 links | counts |
| over_explained_but_true: 13/13 als unnatürlich flagged | NC |
| valid_but_sparse: 16/16 als natürlich akzeptiert | NC |
| natural_but_false: 8/12 (67%) als natürlich akzeptiert | NC |
| weird_marker_free: 0/13 detektiert | NC |
| v3.14 valid manifold survival: 97% | heldout_survival |
| v2.3 valid false-flag: 37% (over-marker-loaded) | per-corpus |

### Drei Lesarten der Performance/Reasoning-Schwelle

1. **Performance ist statistisch erkennbar.** Die 13 OET-Cases
   (über-erklärt aber wahr) werden zu 100 % flagged. Der
   Detektor sagt korrekt: "diese Texte sind zu marker-dicht für
   natürliche kausale Sprache". Die Conclusion ist wahr, aber
   die Form ist Performance. Das ist die ehrliche
   Beobachtung — Wahrheit und Form sind unabhängig messbar.

2. **Sparsity ist die Naturalness-Heuristik.** Die 16 VBS-Cases
   (terse 2-3-sentence valide Chains) werden zu 100 % akzeptiert.
   Natürliche kausale Sprache trägt wenig Marker — sie sagt
   "X happened. Y happened. Therefore Z." ohne Authority-Verben,
   Quantor-Synonyme oder Metaphor-Identifizierungen.

3. **Marker-Free Chaos bleibt unsichtbar.** Die 13 WMF-Cases
   (Non-Sequiturs ohne Marker) werden zu **0 %** detektiert.
   Ihre Feature-Verteilung ist identisch mit valid sparse:
   marker_density = 0, explicitness = 0, alle Marker-basierten
   Features bei Null. Der Detektor sieht keinen Unterschied.

### Die operative Schwelle

> **Explanation wird Performance an dem Punkt, an dem die
> Marker-Dichte den natürlichen Sparsity-Bereich verlässt —
> und sie bleibt Explanation, solange die Marker sparsam und
> domain-kohärent sind. Die Daten zeigen: marker-bearing
> Adversarials (67 % v3.15-Erkennung) sind erkennbar als
> Performance; marker-free Non-Sequiturs (0 % WMF-Erkennung)
> sind als Performance *strukturell* nicht meßbar.**

### Warum das Gate fällt

Vier der fünf Gates fallen (valid_accept, adv_detection,
false_alarm, contamination). Nur heldout_survival hält.

Das ist nicht das Versagen des Probes — es ist die ehrliche
Schwelle des Marker-Paradigmas:

* **v2.3 false-flag (37 %):** v2.3-Chains sind reicher (3+
  Sätze, klare Therefore-Strukturen) als v3.14 sparse Chains.
  Der v3.14-Manifold sieht v2.3 als "zu marker-dicht" und
  flagged sie. Beide sind valid; sie schreiben in
  unterschiedlichen Registern.
* **v3.15 detection (67 %):** Manche v3.15-Angriffe (E
  frame-switch, H non-sequitur) haben keine Marker — sie
  schlüpfen durch. Andere (A, B, C, D, F, G) tragen Marker
  und werden gefangen.
* **WMF NCs (0 %):** Reine Non-Sequiturs ohne Marker sind
  per-feature identisch mit valid sparse — die Methode kann
  sie nicht trennen.

### Was nicht behauptet wird

* **Nicht:** Naturalness-Detection ist nutzlos. Sie fängt 100 %
  der over-explained Performance ab, was eine echte Klasse
  von Manipulation ist.
* **Nicht:** v2.3 chains sind "weniger natürlich" als v3.14.
  Sie haben einen anderen Schreibstil. Eine ehrliche Heuristik
  bräuchte eine Multi-Manifold (per Register).
* **Nicht:** Marker-free Non-Sequiturs sind unentdeckbar.
  Sie sind nur unentdeckbar *mit dieser Klasse von Features*.
  Ein semantischer Embedding-Vergleich würde sie erfassen.

### Die ehrliche Konsequenz

> **Naturalness-Detection auf Marker-Features ist eine
> brauchbare Schicht für *erklärungs-überladene* Manipulation
> (over-explained), aber blind für *erklärungs-arme*
> Manipulation (marker-free non-sequitur). Die zwei
> Manipulations-Klassen brauchen verschiedene Werkzeuge — die
> Performance-Detection deckt eine ab, die andere bleibt für
> semantische Methoden reserviert. v3.18 macht diese
> Aufteilung numerisch ablesbar.**
