## Question

> What remains when surface markers are exhausted?

v4.3 patched 119 of the 143 v4.2 false-supports via a
contamination-safe extension of three closed marker buckets.
The remaining 24 cases survive every existing guard *and*
every marker family v4.3 added. v4.4 asks what kind of
reasoning failure they actually are — and which of five
plausible semantic interventions could rescue them without
breaking the protected benchmarks.

## Method

* **Case selection:** the 24 chains in the v4.0 corpus that
  still audit as `LOGICALLY_SUPPORTED + CAUSAL_CHAIN` under
  the v4.3-patched ``LogicalAuditor``. Count matches the
  v4.3 effect report (`false_support_after = 24`).
* **Replay:** each case is run through the frozen v3.13
  stack; the record carries `frame_kind`,
  `frame_tension_state`, `routing_decision`, `support_state`,
  `premise_count`, per-premise overlap with the conclusion,
  `conclusion_tokens`, `support_path`, and the
  `originating_strategy` set (which of F1–F4 unlocks the
  chain).
* **Classification:** a deterministic priority cascade
  assigns each case *exactly one* `ResidualSemanticFailure`
  value (9 closed values).
* **Counterfactual probes (S1–S5):** five *simulations* of
  alternative routing/audit behaviour. None of them patch the
  runtime; each is a predicate evaluated on every case (and
  every protected VALID chain) to measure rescue rate vs
  contamination risk.
* **Negative controls:** 61 hand-authored semantic fixtures —
  20 cycle disguises, 21 frame switches, 20 non-sequiturs —
  each labelled with the v4.4 class the classifier should
  emit. NC classification accuracy verifies the classifier
  doesn't merely memorise the 24 residue.

## Recommendation

**`SEMANTIC_FAILURE_LOCALIZED`** — the 24 residual
false-supports are not diffuse noise. They concentrate into
four named families (SEMANTIC_SCOPE_COLLAPSE 9,
BIDIRECTIONAL_CYCLE 5, CONCLUSION_LEAP 5,
UNJUSTIFIED_GENERALIZATION 5) with a per-domain signature.
Only one safe semantic probe exists for the v4.4 residue —
S5 covers the 5 BIDIRECTIONAL_CYCLE cases. The remaining 19
require interventions outside the v4.4 scope (e.g.,
semantic NLI or a content-level audit module).

## Reflection

The directive asks: *"When surface evidence is exhausted,
what kind of reasoning failure remains?"*

Using only counts, rescue rates, contamination, and
agreement:

* **4 named clusters** absorb 24/24 cases; the residue is
  not diffuse (entropy = 1.945 bits, four equiprobable-ish
  families).
* **0 UNKNOWN** — the closed semantic taxonomy fully covers
  the surface-marker-exhausted residue.
* **1 safe probe** out of 5 simulated. Rescue rate of the
  one safe probe: 5/24 = 20.8% (BIDIRECTIONAL_CYCLE only).
* **3 aggressive probes** each rescue ≥83% of cases but
  contaminate 276–361 protected VALID benchmark chains. Their
  blast radius is the discipline that disqualifies them.
* **Per-domain signature** — the residue is not a single
  failure mode. Scientific prose fails on contradiction;
  legal prose fails on cycle + contradiction; adversarial
  prose fails on incapacity generalisation + future
  projection. Each demands a different audit-level
  capability.

The boundary between marker-extension and semantic
intervention is not stylistic. v4.3 retired everything that
admitted a contamination-zero token. v4.4 confirms that the
remaining 24 are precisely the cases where the audit would
need a content-level (not surface-level) check, and that
broadening the existing routing without that check
contaminates 11×–15× more valid chains than it rescues.
