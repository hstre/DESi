## Question

> What must be frozen, what must be replayed live, and
> what must be explicitly environment-dependent?

The ZIP spot-audit surfaced two ambiguities the v4 line
had not yet named explicitly:

1. The v1.9 tool benchmark assertion `correct == 20`
   silently assumed a specific SymPy-availability state.
   In an environment where the assumption flipped the
   test failed without naming SymPy.
2. The v2.8 RulePatchProtocol live reconstruction could
   produce a hash other than the canonical
   `1f4d9dfe44cb16e1`. The pin was correct; the
   live-replay path was conflating two distinct claims:
   *the frozen artifact contains hash X* and *the current
   runtime reproduces hash X*.

v4.11 hardens both by replacing the implicit assumptions
with explicit policy + classification — every artifact
receives one closed `ReproducibilityClass` value, and
every test declares which class of claim it is asserting.

## Method

* **Environment fingerprint** (`environment.py`) — captures
  python version, platform, optional-dependency
  availability + versions, git commit + branch, and the
  relevant environment variables. Deterministic per
  environment. Written to
  `artifacts/v4_11/environment.json`.
* **Tool repro policy** (`tool_policy.py`) —
  `TOOL_REPRO_POLICY = ENVIRONMENT_CONDITIONAL`. Closed
  expected-correct table:
    - SymPy absent → 20/20,
    - SymPy present → 16/20.
* **Replay matrix** (`replay_matrix.py`) — for each
  required version (v2.8, v3.11-v3.23, v4.0-v4.10) record
  artifact_exists, frozen_hash, live_replay_available,
  live_hash, hash_equal, repro_class. Written to
  `artifacts/v4_11/replay_matrix.json`.
* **Closed reproducibility taxonomy**
  (`ReproducibilityClass`):
    - `FROZEN_ARTIFACT_REPLAYABLE`,
    - `LIVE_REPLAY_STABLE`,
    - `ENVIRONMENT_DEPENDENT`,
    - `HISTORICAL_RUNTIME_DRIFT`,
    - `NON_REPLAYABLE_BY_DESIGN`,
    - `UNKNOWN`.
* **Negative controls** — twenty closed
  (artifact_exists, live_replay_available, hash_equal,
  frozen_only) tuples mapped to their expected class.

## v2.8 replay boundary

| field | value |
|---|---|
| `v2_8_frozen_hash` | `1f4d9dfe44cb16e1` |
| `v2_8_live_hash` | `1f4d9dfe44cb16e1` (this environment) |
| `v2_8_hash_equal` | `True` |
| `v2_8_repro_class` | `LIVE_REPLAY_STABLE` |

In an environment where the v2.8 live replay returns a
different hash, the same matrix call would record
`v2_8_hash_equal = False` and
`v2_8_repro_class = HISTORICAL_RUNTIME_DRIFT`. The frozen
artifact pin (`1f4d9dfe44cb16e1`) is always asserted from
disk; the live hash is reported separately. No test claims
that the two are equal unless the matrix declares
`LIVE_REPLAY_STABLE`.

## Tool benchmark policy

Environment-conditional expected count:

```python
def expected_correct_count(sympy_available):
    return 16 if sympy_available else 20
```

The four `B_symbolic_math` cases (TB1-TB4) carry ground
truth `SHOULD_TOOL_FAIL`. With SymPy absent the tool
correctly fails (TOOL_FAILED, DEPENDENCY_MISSING) and the
case matches ground truth. With SymPy present the tool
succeeds (TOOL_SUPPORTED) against a `SHOULD_TOOL_FAIL`
ground truth and the case grades incorrect. The test
`test_live_tool_benchmark_matches_policy` verifies the
live count against the policy's prediction for the
current environment.

## Replay matrix

| class | versions |
|---|---|
| `LIVE_REPLAY_STABLE` (2) | v2_8, v4_9 |
| `FROZEN_ARTIFACT_REPLAYABLE` (12) | v3_11 ... v3_23 |
| `HISTORICAL_RUNTIME_DRIFT` (9) | v4_0, v4_1, v4_2, v4_3, v4_4, v4_5, v4_6, v4_7, v4_8 |
| `NON_REPLAYABLE_BY_DESIGN` (1) | v4_10 |
| `UNKNOWN` (0) | — |

The nine `HISTORICAL_RUNTIME_DRIFT` entries are the
documented downstream effect of the v4.3/v4.5/v4.7/v4.9
runtime patches: each v4.0-v4.8 report was authored
against an earlier auditor state; the live rebuild under
the current (v4.9-patched) auditor produces a different
hash. The drift is *explicit* — surfaced in the matrix,
named with a closed class, and not silently hidden behind
a passing test.

## Negative controls

hash_equal, frozen_only)` tuples cover all six
`ReproducibilityClass` values plus the boundary

## Test changes

No test in the v4.11 module claims *live replay equals
historical artifact* unless the corresponding matrix entry
declares `LIVE_REPLAY_STABLE`. Existing tests in
`tests/tools/test_benchmark.py` are unchanged; they
already gate the SymPy-absent assertion with
`pytest.skip` (see `test_b_cases_dependency_missing`). The
v4.11 test
`test_live_tool_benchmark_matches_policy` is the new
environment-aware contract.

## Reflection

The directive asks: *"When is reproducibility a live
computation — and when is it a preserved historical
artifact?"*

Using only the v4.11 matrix + environment fingerprint:

* **Two** versions are *live computations*:
  v2.8 reconstruction (under the current runtime) and
  v4.9 (the most recent runtime patch). Both produce
  hash-equal-to-frozen.
* **Twelve** versions are *preserved historical
  artifacts*: v3.11 through v3.23. No live builder is
  exposed; their canonical record is the artifact file.
* **Nine** versions are *historical artifacts with
  documented drift*: v4.0-v4.8. The frozen file is the
  v4.x-era record; the live rebuild produces a different
  hash under subsequent runtime patches; the matrix
  records both halves.
* **One** version is *non-replayable by design*: v4.10
  (the consolidation paper artifact, assembled via a
  one-off generator script).
* **Zero** versions are `UNKNOWN`.

Reproducibility is a live computation iff and only iff
the runtime that produced the artifact still equals the
current runtime. Reproducibility is a preserved
historical artifact iff the artifact file remains
canonical regardless of runtime drift. v4.11's discipline
is to refuse to conflate the two — every artifact
receives exactly one closed class, and every test
declares which class of claim it makes.
