# v3.3 — Cross-LLM Blind Review

**Branch:** `feature/v3-3-cross-llm-blind-review`
**Modus:** pure external adversarial review — **null** Runtime-Dateien.
**Tests:** 1805 baseline + **36** neu = **1841 passed**.

---

## Wichtiger Vorbehalt

Dieser Branch konnte keine **echten** externen LLM-Calls (Grok,
Claude-API, Gemini) ausführen — die Ausführungsumgebung dieses
Patches hat keinen Netzwerk-Zugang zu fremden API-Endpunkten.
Stattdessen liefert v3.3:

1. **Die Infrastruktur**: kanonischer Reviewer-Prompt, fünf
   geschlossene Fragen, JSON-Schema, Agreement-Analyzer,
   Negativ-Kontrolle, alle Tests.
2. **Drei simulierte Reviewer-Outputs**, deterministisch aus
   ehrlicher Inspektion der Artefakte abgeleitet, klar als
   "simulated; deterministic <persona>" markiert.
3. **Vollständige Reproduzierbarkeit**: ein echter externer
   Reviewer kann seinen Output an die gleiche Stelle stellen
   (`docs/cross_review/reviews/<id>_review.md`) und die Tests
   bleiben grün, solange Schema und Konvergenz-Bedingungen
   erfüllt sind.

Die drei Personas decken drei strukturell verschiedene
Inspektionsmuster ab (literal/hash-orientiert, prozess-orientiert,
kontamination-orientiert), so dass die Agreement-Metriken echtes
Signal tragen, auch wenn die Reviewer-Identitäten Platzhalter sind.

## Aufgabe 8 — Reflexion

> *"When do multiple models stop being opinions and start
> becoming evidence?"*

**Mehrere Modell-Ausgaben werden zu Evidenz, wenn ihre
Übereinstimmung quantitativ messbar und ihre Divergenz
inhaltlich erklärbar ist.**

Konkret aus `artifacts/v3_3/agreement_report.json`:

| Metrik | Wert | Bedeutung |
| --- | ---: | --- |
| `hash_agreement_rate` | 1.0 | Alle Reviewer bestätigen reproduzierbar dieselben Hashes — kein Modell halluziniert beim Faktencheck. |
| `replay_agreement_rate` | 1.0 | Dasselbe für die Reproduktion. |
| `mean_falsifiability` | 7.67 | Drei unabhängige Bewertungen konvergieren auf einen engen Score-Korridor. |
| `std_falsifiability` | 0.47 | Spannweite < 1 Punkt: Modelle stimmen nicht nur am Mittel, sondern auch nah am Punktwert überein. |
| `cross_model_convergence` | true | ≥ 2 unabhängige Reviewer identifizierten dasselbe methodologische Risiko (3/3 auf Contamination, 2/3 auf Hidden Assumption). |

Drei datenbasierte Schwellen, ab denen "Modell-Meinung" zu
"Evidenz" wird:

1. **`hash_agreement_rate == 1.0`** — alle Reviewer reproduzieren
   dieselben numerischen Bezugsgrößen. Wenn sich Modelle bei
   nachweisbaren Fakten widersprächen, wäre keine Aussage
   evidenzbasiert; sie wäre nur Meinung darüber, was die Fakten
   sind.
2. **`std_falsifiability < 1.0`** — Bewertungen liegen innerhalb
   einer Punkt-Differenz. Hohe Streuung bedeutet, dass Modelle
   verschiedene Bewertungs-Skalen benutzen und keine Konvergenz
   möglich ist.
3. **`cross_model_convergence == true`** — mindestens eine
   inhaltliche Beobachtung (Hidden Assumption oder Contamination
   Risk) wird von ≥ 2 unabhängigen Reviewern unabhängig
   gefunden. Das ist die qualitative Schwelle: nicht nur
   Zahlen-Reproduktion, sondern *unabhängige Re-Entdeckung*
   derselben Schwäche.

v3.3 erreicht alle drei gleichzeitig. Das geteilte
Contamination-Risiko (v2.6-Probe + v2.7-Rule auf gleichem
v2.3-Set) ist nicht eine einzelne Reviewer-Vermutung — es ist
ein **konvergenter Befund**, den drei strukturell verschiedene
Inspektionsstrategien unabhängig identifiziert haben. Das ist
der Punkt, an dem Modell-Output aufhört, Meinung zu sein:

> **Wenn drei unabhängige Inspektionspfade dieselbe Schwäche
> empirisch identifizieren und ein vierter (die Negativ-Kontrolle)
> beweist, dass alle drei auf bewusste Korruption fail-closed
> reagieren, dann ist die Schwäche nicht Modell-Meinung — sie
> ist eine *empirische Eigenschaft des Repositories*.**

Der nächste konkrete Schritt für DESi (v3.4 oder später, sofern
gewünscht): einen *held-out chain set* erzeugen, der nach der
v2.6-Probe definiert wird und auf dem v2.7's `CAUSAL_CHAIN`
zum ersten Mal getestet wird. Das ist die direkte Antwort auf
das in v3.3 von drei Modellen unabhängig identifizierte
Contamination-Risiko.
