## Question

> Why does `CAUSAL_CHAIN` accept invalid reality once routing
> is unlocked?

v4.1 showed that synthetic frame ingress through the v3.13
router unmasked 143 invalid external chains for which the
unmodified `LogicalAuditor` returned
`LOGICALLY_SUPPORTED` under `CAUSAL_CHAIN`. v4.2 asks: are
those 143 cases scattered noise, or do they cluster into a
small number of named failure modes?

## Method

* **Case selection:** every chain in the v4.0/v4.1 corpus
  with `ground_truth = INVALID` for which the unmodified
  auditor returns `LOGICALLY_SUPPORTED + CAUSAL_CHAIN`. Count:
  **143**, matching the v4.1 instrumentation finding.
* **Replay:** each case is run through the frozen stack
  (`SPL → FrameDeclaration → FrameTension →
  FrameTensionRouter → CAUSAL_CHAIN → SuspensionGate`) and
  every observable surface artefact is recorded, including the
  v4.1 `frame_strategy_origin` (which of F1–F4 unlocks the
  case).
* **Classification:** each case receives *exactly one*
  `ExternalAuditFailure` value via a deterministic
  priority cascade (METAPHOR > AUTHORITY > HIDDEN_NEGATION >
  QUANTIFIER_DRIFT > TOOL > CYCLE_DISGUISE > FRAME_SWITCH >
  EXTRACTION_COLLAPSE > SEMANTIC_NON_SEQUITUR > UNKNOWN).
* **Counterfactual:** for each cluster, the surface tokens
  observed in its cases are simulated as candidate extensions
  of the corresponding v3.16 marker bucket. Reduction (how
  many false-supports would flip to REJECTED) and contamination
  (how many currently-passing benchmark cases would break) are
  measured *without patching runtime*.
* **NCs:** 50 synthetic external false-supports
  (`src/desi/external_audit_probe/negative_controls.py`),
  five per surface-bearing class plus five UNKNOWNs — every
  closed class is covered. Classifier accuracy on this set is
  asserted to reach `>= 0.95`.

## Guard pressure

* `actual_firing_rate = 0.000` (every case passes every
  existing guard — definitional, since the chains are *false

* `shadow_firing_rate = 0.587` — 84/143 cases carry at least

* `cases_with_zero_guard_hits = 59` — the remaining 41% have no
  recognisable v2.7/v3.16-adjacent surface token; their
  failure mode is structural (FRAME_SWITCH, SEMANTIC, CYCLE) or

## Recommendation

**`AUDIT_FAILURE_LOCALIZED`** — the 143 v4.0 false-supports are
not diffuse noise. They concentrate into a small number of
named families dominated by `HIDDEN_NEGATION` (48%) and
`QUANTIFIER_DRIFT` (25%). Three of the four largest clusters
(HIDDEN_NEGATION, QUANTIFIER_DRIFT, AUTHORITY_CONTAMINATION)
admit zero-contamination extensions of the existing v3.16
marker buckets that would, together, retire 119/143 = 83% of
the false supports without touching any other guard.

## Reflection

The directive asks: *"When does a reasoning rule fail because
the world is ambiguous — and when because the rule cannot see
what the world is saying?"*

The counts answer with no rhetoric.

* **The rule cannot see what the world is saying:**
  119 cases (HIDDEN_NEGATION + QUANTIFIER_DRIFT +
  AUTHORITY_CONTAMINATION) carry a single surface token the
  existing v3.16 buckets do not list — a finite, surface-clean,
  closure-safe gap.
* **The world is genuinely ambiguous:**
  19 cases (FRAME_SWITCH + SEMANTIC_NON_SEQUITUR) carry no
  recognisable surface marker at all; the conclusion's
  invalidity lives in content the v2.7 `CAUSAL_CHAIN` rule has
  no semantic mechanism to inspect.
* **5 cases (CYCLE_DISGUISE)** are circular at the content-
  token layer but slip just under the v2.7 recycled-conclusion
  guard's 2-overlap threshold.

The audit's external-prose blind spot is therefore *mostly*
mechanical: 83% sits in a closed marker gap whose closure
carries zero benchmark contamination. The residual 13% is
semantic and is the genuine open problem for a future
audit-level upgrade — outside the v4.2 scope.
