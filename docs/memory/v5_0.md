# v5.0 — Methodology Transfer Probe

**Status:** read-only audit. No runtime patches, no v4
taxonomy reuse, no benchmark changes.
**Branch:** `claude/init-desi-prototype-2QjHF`.
**Artifact:** `artifacts/v5_0/report.json`,
`artifacts/v5_0/taxonomy.json`.
**Tests:** 62 new under `tests/methodology_transfer/`;
full v5.0 suite green.

## Question

> When DESi is shown five domains it has never audited,
> can it re-run its own v4 method — failure localisation,
> closed taxonomy, safe-probe identification, contamination
> audit, narrow patch candidate, explicit recommendation —
> without manual taxonomy injection and without copying any
> v4 class name?

The v4 line iteratively built its taxonomy by hand against
five seen domains. v5.0 tests whether the *method* survives
when the *content* changes. Forbidden: importing any v4
taxonomy, reusing any v4 probe id, modifying any v4 runtime
file.

## Method

* **Corpus (Aufgabe 4)** — 565 chains across five new
  domains: D1 `technical_incident_reports` (120),
  D2 `legal_case_summaries` (125), D3 `medical_guidelines`
  (110), D4 `scientific_peer_reviews` (105),
  D5 `mathematical_proof_sketches` (105). Hand-authored
  prose; balanced VALID 185 / INVALID 185 / AMBIGUOUS 195
  (within ±20% of mean).
* **Feature extraction (Aufgabe 5)** — closed 18-feature
  schema in `FEATURE_NAMES`: premise count, conclusion
  novelty ratio, premise-conclusion overlap counts, modal /
  negation / universal surface presence, ed-suffix verb
  cues, audit polarity, ground-truth one-hot. None of the
  names matches any v4 marker bucket.
* **Failure discovery (Aufgabe 5)** — 346 of 565 chains
  audit as failures (audit polarity disagrees with ground
  truth). Per-domain failure rates: D1 0.467, D2 0.680,
  D3 0.591, D4 0.762, D5 0.571.
* **Cluster discovery (Aufgabe 6)** — deterministic
  single-pass agglomerative clustering on the feature
  vectors with L1 tolerance 1.5. No k-means randomness.
  Tail collapsed into MT_OTHER cap at twelve.
* **Taxonomy naming (Aufgabe 6)** — closed cascade in
  `TaxonomyClass`, eight classes assigned: every value
  carries the new `MT_` prefix. The cascade evaluates
  `MT_AMBIGUITY_DECISIVENESS` first (label-restricted) so
  hedge tokens do not collide with strong modals;
  `MT_OVERLAP_LOOP` before `MT_UNIVERSAL_LEAP` so cyclic
  conclusions whose phrasing carries "every" stay in their
  structural class.
* **Probe generation (Aufgabe 7)** — one candidate probe
  per taxonomy class, IDs `MT-P01..MT-P09`. Predicates
  tightened per v4-line lessons: `_STRICT_MODALS =
  {will, cannot}`, `_STRICT_NEGATION = {excluded, denied,
  withheld}`, `_STRICT_UNIVERSALS` is multi-word only
  (`every patient`, `for every`, etc.).
* **Contamination audit (Aufgabe 8)** — protected pool =
  every chain in MAIN/MULTISTEP/HELDOUT/ADVERSARIAL/TOOL
  benchmarks plus v4.0 external VALID chains, filtered to
  those that currently audit as `LOGICALLY_SUPPORTED`
  (mirroring the v4.5/v4.7/v4.9 contract). Pool size 385.
  A probe is safe iff contamination == 0.
* **Patchability (Aufgabe 11)** — `PATCHABLE` when the
  probe is safe and rescues ≥ 1 cluster member,
  `UNPATCHABLE` when unsafe, `AMBIGUOUS` when safe but
  inert against observed failures.
* **Negative controls (Aufgabe 10)** — 101 NCs across six
  families (MA, NA, UL, OL, NE, AD); classifier accuracy
  1.000.
* **Recommendation gate (Aufgabe 12)** — closed cascade
  over six thresholds; all must pass for `CONFIRMED`.

| MT-P05 | `MT_NOVEL_ENTITY` | ✗ | 298 | 77 | UNPATCHABLE |
| MT-P06 | `MT_AMBIGUITY_DECISIVENESS` | ✗ | 15 | 190 | UNPATCHABLE |

| MT-P08 | `MT_AUDIT_OVER_BLOCK` | ✓ | 0 | 0 | AMBIGUOUS |

The two `UNPATCHABLE` results are expected: a 70%-novel
content threshold and a hedge-token surface check both
match many genuinely valid chains. v5.0 does not patch
either — the contamination audit *correctly rejected* them
as runtime candidates.

## Recommendation

**`METHODOLOGY_TRANSFER_CONFIRMED`** — every gate
satisfied: cluster count in [5, 12]; unknown fraction = 0;
largest cluster ≥ 15%; ≥ 3 safe probes (six of eight);
NC accuracy ≥ 0.95; taxonomy completeness ≥ 0.90.

## What this licenses (and what it does not)

* The v4 *method* — feature extraction → deterministic
  clustering → closed-cascade naming → predicate generation
  → protected-pool contamination — transfers to five
  unseen domains without manual seeding.
* It does **not** license any runtime patch. v5.0 ships
  zero changes to `src/desi/logic`, `frames`,
  `frame_tension`, `frame_inference`, `recursive`,
  `consilium`, `tools`, `benchmark*`, or any artifact under
  prior version directories.
* It does **not** demonstrate that every domain produces
  equally rich taxonomies: cross-domain transfer variance
  is small (0.0101) but failure rates still differ by
  ~30 percentage points across the five domains.
