# DESi v0.4.1 — First Observable Epistemic Showcase

v0.4.1 takes three already-validated scenarios from the v0.4 set
(S2 contradictory_claims, S6 refused_merge, S7 memory_trap) and turns
each one into a bundle of artefacts a reviewer can read directly.

Design principle:

> Nicht behaupten, dass DESi anders denkt. Zeigen, dass DESi anders
> prüft.

v0.4.1 does not change DESi behaviour. It re-uses the v0.4
EvaluationHarness unchanged and adds an emit-only layer on top.

## New surface

```
src/desi/showcase/
  __init__.py          public API
  descriptions.py      hand-crafted prose per showcase scenario
  runner.py            ShowcaseRunner + ShowcaseArtifacts
```

## ShowcaseRunner

```python
from desi.showcase import ShowcaseRunner

runner = ShowcaseRunner(out_dir="docs/showcase")
bundles = runner.run_all(seed=42)
# {
#   "S2": ShowcaseArtifacts(directory=docs/showcase/S2, ...),
#   "S6": ShowcaseArtifacts(directory=docs/showcase/S6, ...),
#   "S7": ShowcaseArtifacts(directory=docs/showcase/S7, ...),
# }
```

Per-scenario artefacts (seven files):

| file                    | content                                         |
|---                      |---                                               |
| `summary.json`          | evaluation id, seed, model, config hash, passes |
| `timeline.md`           | tick-by-tick timeline (markdown table)          |
| `timeline.json`         | same timeline, JSON                             |
| `snapshot_start.json`   | graph state at run start                        |
| `snapshot_end.json`     | graph state at run end                          |
| `snapshot_end.cypher`   | end-state as idempotent Neo4j MERGEs            |
| `analysis.md`           | Problem / Verhalten / Endzustand / Warum        |

Plus one top-level `baseline_notes.md` that contrasts the classical
LLM path with the DESi path across all three scenarios. No marketing
language; the comparison describes only what is structurally visible
in the artefacts.

## What v0.4.1 deliberately does NOT do

Explicit non-goals from the directive:

- no new operators
- no memory reads inside the run
- no guard adaptation
- no self-learning, no ranking
- no embeddings
- no UI gimmicks
- no web server
- no notebook dependencies

## The three showcase scenarios

| ID  | Title                  | Demonstrated property                        |
|---  |---                     |---                                            |
| S2  | Contradiction Detection| bidirectional `CONTRADICTS` edges in the graph |
| S6  | False Merge Rejection  | `guard_blocked` event + no `MERGED_INTO` edge |
| S7  | Memory Trap            | legacy and new claims coexist with distinct methods |

## Determinism

`ShowcaseRunner.run_all(seed=42)` produces:

* `timeline.md` byte-identical between runs (no wall-clock fields)
* `snapshot_*.json` structurally identical (wall-clock + UUID fields
  stripped before comparison)
* `summary.json` structurally identical (per-run `evaluation_id` and
  ISO timestamp documented as out-of-scope for determinism)
* `baseline_notes.md` byte-identical (derived from hand-written
  descriptions only)

Tested by
`tests/showcase/test_runner.py::test_same_seed_produces_structurally_identical_artifacts`
and `::test_baseline_notes_is_deterministic_between_runs`.
