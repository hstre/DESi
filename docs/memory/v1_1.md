# DESi v1.1 â€” Real LLM + Source Documents

v1.1 is **the first real-sensor release**. v1.0 showed that DESi can
ingest natural language without losing epistemic discipline. v1.1
makes the input real: structured `SourceDocument`s with MIME types,
a four-format parser (plain / markdown / HTML / PDF), a real
DeepSeek client with timeout / retry / size guards, and per-document
cost budgets.

Design principle:

> The LLM remains a semantic sensor.
> Not a memory authority.
> Not a merge authority.
> Not a contradiction authority.
> Not a jury authority.
> Not a mutation authority.
> Not a promotion authority.

## SourceDocument â€” first-class structured input

```python
SourceDocument(
    document_id="doc_abc",
    content="Water boils at 100Â°C at sea level.",
    content_type="text/plain",
    author="alice",
    language="en",
    created_at=â€¦,
)
```

Construction-time checks:

* `document_id` non-empty
* `content_type` non-empty
* `text/*` types require `str` content
* `application/pdf` requires `bytes` content

The supported MIME types are a **closed** set:

```
text/plain
text/markdown
text/html
application/pdf
```

Anything else fails-closed at the parser with kind
`unsupported_document_type`. `to_dict()` deliberately excludes the
content body (payloads may be large or sensitive); only metadata
appears in audit dumps.

## DocumentParser â€” four formats, pure stdlib

`DocumentParser.parse(doc)` returns a `ParseResult` with:

* `normalized_text` â€” the canonical text fed into the SemanticBackend
* `parser_version` â€” pinned to `"v1.1"`
* `parser_hash` â€” `"pa_"` + sha256-16 of the normalised text
* `document_id`, `content_type` â€” propagated

Per format:

| Type              | Strategy                                                                |
|---                |---                                                                       |
| `text/plain`      | whitespace-normalise; empty input â†’ `empty_document`                     |
| `text/markdown`   | remove headings / `**bold**` / `*italic*` / inline `` `code` `` / fenced |
|                   | code / `[link](url)` / list and ordered bullets                          |
| `text/html`       | drop `<script>` / `<style>` / `<noscript>` blocks; strip remaining tags; |
|                   | decode HTML entities; insert newlines on block tags                      |
| `application/pdf` | parse BT / ET text blocks; decode literal `(...) Tj` operators; support  |
|                   | `/Filter /FlateDecode` via stdlib `zlib`; no fonts, no OCR, no images    |

A PDF that lacks the `%PDF` header or produces an empty text body
fail-closes with `pdf_parse_failed` / `empty_document`.

## DeepSeekClient â€” the only network sensor

```python
DeepSeekClient(
    api_url=DEEPSEEK_API_URL,            # default https://api.deepseek.com/...
    model_id="deepseek-4-pro",            # pinned
    temperature=0.0,                       # caller-tunable
    max_tokens=512,                        # caller-tunable
    timeout=10.0,                          # hard, not configurable for prod
    max_retries=2,                         # pinned
    retry_backoff=(2.0, 4.0),              # exponential
    transport=â€¦,                           # injectable (tests use a fake)
)
```

Network rules (pinned in code):

* **Key** is read **exclusively** from the environment variable
  `DEEPSEEK_API_KEY`. The client refuses to construct a request when
  the variable is unset. No alternative source (config file, ledger,
  CLI flag) exists.
* **Timeout**: 10 seconds per attempt.
* **Retries**: 2 (so up to 3 total attempts) with exponential backoff
  `(2 s, 4 s)`. The backoff is driven by an injectable `sleep_fn` so
  tests run instantly.
* **5xx HTTP** â†’ retry. **4xx HTTP** â†’ no retry (the request itself
  is wrong; retrying would not help).
* **Response size**: hard cap at 50 KB (`MAX_RESPONSE_SIZE_BYTES`).
  Anything bigger raises `BackendError("response_too_large")`.
* **Encoding**: UTF-8; non-UTF-8 â†’ `BackendError("invalid_encoding")`.

The key is **never stored** on the instance. The ledger never
contains the key. The client is a thin shim around `urllib.request`
with no global state.

`DeepSeekClient` is callable: `client(prompt) -> str` matches the
`LLMSemanticBackend.llm_call` signature exactly, so wiring is:

```python
adapter = SPLAdapter(
    backend=LLMSemanticBackend(llm_call=DeepSeekClient()),
)
```

## Per-document cost guard

v1.0 budgeted **20** LLM projections per adapter. v1.1 adds a
second budget: **5** LLM projections per `document_id`. The 6th
projection of the same document fail-closes with
`cost_guarded=True` and a `SEMANTIC_PROJECTION_REJECTED { reason:
"per_document_budget_exhausted" }` ledger event.

The two budgets compose:

* 4 documents Ă— 5 calls = 20: per-adapter budget hits exactly.
* 5 documents Ă— 5 calls = 25: 5 calls per-document admitted, then
  per-adapter budget caps at 20 total admissions.

The deterministic backend is never per-document tracked â€” it costs
nothing to run.

`adapter.projections_for_document(doc_id) -> int` exposes the
current count for audit.

## Fail-closed semantics (v1.1 additions)

| Failure                          | Kind                          | Ledger event              |
|---                               |---                            |---                         |
| Missing `DEEPSEEK_API_KEY`       | `missing_api_key`             | `LLM_REQUEST_FAILED`       |
| `TimeoutError` (Ă— retries)       | `timeout`                     | `LLM_REQUEST_FAILED`       |
| `URLError` (Ă— retries)           | `url_error`                   | `LLM_REQUEST_FAILED`       |
| `HTTPError` 5xx (Ă— retries)      | `http_5xx`                    | `LLM_REQUEST_FAILED`       |
| `HTTPError` 4xx (no retry)       | `http_4xx`                    | `LLM_REQUEST_FAILED`       |
| Response > 50 KB                  | `response_too_large`          | `LLM_RESPONSE_REJECTED`    |
| Invalid JSON                      | `invalid_json`                | `LLM_RESPONSE_REJECTED`    |
| Wrong JSON schema                 | `invalid_schema`              | `LLM_RESPONSE_REJECTED`    |
| Empty output                      | `empty_output`                | `LLM_RESPONSE_REJECTED`    |
| Unsupported document type         | `unsupported_document_type`   | `SEMANTIC_PROJECTION_REJECTED` |
| Empty / un-parseable document     | `empty_document`              | `SEMANTIC_PROJECTION_REJECTED` |
| PDF without `%PDF` header         | `pdf_parse_failed`            | `SEMANTIC_PROJECTION_REJECTED` |
| Per-doc budget exhausted          | (sentinel)                    | `SEMANTIC_PROJECTION_REJECTED` |
| Per-adapter budget exhausted      | (sentinel)                    | `SEMANTIC_PROJECTION_REJECTED` |

No backend exception ever crosses the adapter boundary. Every error
yields zero claims plus a ledger entry.

    return giant

## Did DeepSeek run cleanly under quarantine?

**Yes.** Four belts-and-braces:

1. **Key isolation.** The key is read from `DEEPSEEK_API_KEY` only.
   It is never stored on the client instance, never logged, and
   never present in any ledger payload (`test_d8_no_key_does_not_leak_anywhere_into_ledger`
   inspects every entry's serialised dict for the substring `sk-` and
   `DEEPSEEK_API_KEY`).
2. **Injectable transport.** `DeepSeekClient(transport=â€¦)` lets the
   entire test suite drive the timeout / retry / 4xx / 5xx / oversize
   paths without making a single real network call. The default
   transport (`_urlopen_transport`) is only wired when the caller
   omits the kwarg.
3. **Fail-closed boundary.** Every error path inside the client
   raises `BackendError(kind=â€¦)`. The `SPLAdapter` catches that
   single exception type, logs `LLM_REQUEST_FAILED` or
   `LLM_RESPONSE_REJECTED` based on `error.category`, and returns a
   `SPLProjectionResult` with `claims=()`. No backend exception ever
   propagates into memory / observe / evolution code.
4. **Per-document budget.** Even when an LLM call works, the
   `PER_DOCUMENT_PROJECTION_BUDGET = 5` cap stops a single
   document from absorbing more than 5 calls. The 6th attempt on the
   same `document_id` is silently dropped (`cost_guarded=True`,
   ledger event written, no `BackendError`, no API call made).

All 573 tests pass without network access. The full v1.0 baseline is
unchanged.

## Open design questions for v1.2

1. **Memory recorder integration.** v1.1 returns `Claim` objects; it
   does not write to memory. v1.2 needs a documented contract for
   how a recorder consumes SPL-produced claims â€” in particular,
   whether `spl_llm`-sourced claims enter a quarantine state until a
   second source (`spl_deterministic`, human, etc.) corroborates.
2. **Multi-document context.** Two `SourceDocument`s with the same
   content produce two distinct `Claim`s (different `run_id`s). v1.2
   should formalise whether DESi's merge operator may eventually
   propose a cross-document merge, and under which evidence rule.
3. **Per-key cost budget.** v1.1 budgets per-adapter and per-document.
   Production deployments will need a per-API-key budget enforced at
   the DeepSeek call site (with a structured `BackendError` kind
   `quota_exceeded`).
4. **Streaming responses.** A 50 KB hard cap is appropriate for
   small projections. v1.2 should consider whether the SPL ever needs
   streamed responses for long documents â€” and if so, how the size
   guard works at the streaming boundary.
5. **Retry policy under load.** Fixed `(2 s, 4 s)` backoff is
   conservative. v1.2 could let the backoff be a callable so that
   production callers can plug in jitter and circuit-breaker logic
   without changing the client.
6. **Multiple LLM providers.** v1.1 hard-codes DeepSeek. v1.2 should
   formalise the "swap model" path: a code edit that updates both
   `DEEPSEEK_MODEL_ID` and a documented compatibility note, and a
   ledger field that records which provider built each Claim.
7. **PDF support depth.** The v1.1 PDF extractor is intentionally
   minimal â€” uncompressed text + FlateDecode, no fonts, no images.
   v1.2 needs a policy: do we add `pypdf` as an optional dependency
   (with the parser falling back if it's not installed) or keep the
   pure-stdlib path as the only supported one?
8. **Audit of prompt text.** Today the prompt is hashed but not
   logged in plain. v1.2 might add an opt-in `prompt_log_dir` so
   reviewers can inspect rendered prompts post-hoc (with the same
   key-leakage protection as the client).
9. **Jury awareness of source quality.** The v0.6 jury still does
   not consume `SPLProvenance`. v1.2 could let the Skeptiker
   downweight evidence with `source == spl_llm` until corroborating
   evidence is recorded.
