## Reflection

The directive asks: *"When does external generalization
stop being a benchmark result — and become an
architectural claim?"*

Using only the counts in the v4.10 artifact:

* **claim_count = 127** over twenty-three pinned versions.
* **failed_hypotheses = 10**, each with concrete artifact
  evidence.
* **23 pinned hashes** preserved bit-identical across all
  four v4-line runtime patches.
* **contamination history**: every runtime patch (v4.3,
  v4.5, v4.7, v4.9) measured contamination 0 on the
  protected pool, individually and cumulatively.
* **final false_support count = 0**: zero v4.0 external
  chains remain misclassified as supported after the v4.9
  runtime patch.

External generalization stops being a benchmark result and
becomes an architectural claim at the point where four
independent localization-and-patch cycles all converge on
the same shape: closed failure taxonomy, safe-probe
identification, contamination-zero runtime guard,
preserved-hash regression. v4.0 -> v4.9 is that shape
instantiated four times. v4.10 is the certification that
all twenty-three pins, all 127 claims, all ten falsified
hypotheses, and all four patches survive together —
without contradiction, without drift, without missing
pins.
