## Aufgabe 7 — Reflexion

> *"When does reproducibility become transferable?"*

**Reproduzierbarkeit wird übertragbar, wenn sie sich
*innerhalb des Repositories selbst* in einem festen Budget aus
Kommandos und Zeit auf einer Maschine, die das Repository zum
ersten Mal sieht, vollständig nachvollziehen lässt — ohne
implizites Wissen aus Chat-Historien, Slack-Threads oder
mündlicher Tradition.**

Drei datenbasierte Schwellen aus
`artifacts/v3_2/reviewer_metrics.json`:

1. **`commands_required ≤ 15`** — Budget für eine Sitzung mit
   einem externen Reviewer. v3.2: **7**. Jeder Schritt im
   Quickstart ist eine einzige Shell-Zeile, kein Skript, keine
   Setup-Notebooks, keine versteckten Abhängigkeiten.

2. **`estimated_minutes ≤ 15`** — kognitives und realzeitliches
   Budget. v3.2: **5**. Über das Limit wird die Reproduktion zur
   Ermüdungsübung; darunter ist sie ein Akt der Verifikation.

3. **`hash_mismatches + broken_links + missing_paths == 0`** —
   strikte Konsistenz zwischen Dokumenten, Claim-Index, Artefakten
   und Tests. v3.2: **0 + 0 + 0 = 0**. Jede Behauptung im Bundle
   ist von einem Test gegen ein Artefakt geprüft; jeder Artefakt-
   Pfad existiert; jeder zitierte Hash deckt sich mit dem Feld im
   Artefakt.

> **Solange einer dieser drei Zahlen versagt, ist
> Reproduzierbarkeit ein Privileg derer, die schon mit dem Projekt
> gearbeitet haben. Wenn alle drei gleichzeitig gelten, hat das
> Repository die Schwelle zur Übertragbarkeit überschritten — die
> nächste Person, die es lädt, hat die gleiche Beweislage wie
> diejenigen, die es gebaut haben.**

Vor v3.2 hätte ein externer Reviewer, der die DESi v2.7/v2.8-
Behauptungen verifizieren wollte, durch zehn `docs/memory/v*.md`-
Dateien lesen, die Test-Suite-Struktur entdecken, die richtigen
`pytest`-Befehle auswählen und die Hash-Werte aus laufenden
Beobachtungen extrahieren müssen. Wahrscheinlich aufgegeben nach
drei Minuten.

Nach v3.2 sind die genau notwendigen sieben Kommandos in
`quickstart.md` aufgelistet, jeder erwartete Output ist
verbatim notiert, und ein Test-Suite-Run beweist, dass das alles
zueinander passt.

> **Reproduzierbarkeit wird übertragbar, wenn das Onboarding-Budget
> für einen Reviewer kleiner ist als das Geduldsbudget eines
> echten Reviewers.**

v3.2 setzt diese Schwelle bei *sieben Kommandos und fünf Minuten*
— deutlich unter der menschlichen Geduldsgrenze, deutlich über
einer Trivialgrenze. Wenn die nächste DESi-Version diese Schranke
überschreitet, wird `reviewer_metrics.json` es im ersten Lauf
sichtbar machen.
