# Generalization loop — n=20 unseen-trajectory suite

Companion to the n=10 self-improvement loop (`experiments/self_improvement/`).
Purpose: test whether the 11 detector/classifier changes from cycles 1-11
**generalize** beyond the original adversarial fixtures, or **overfit** to
them.

## Layout

- `generate_fixtures.py` — emits 20 fixtures into `data/generalization/`.
  Source of truth; the JSONs are deterministic outputs.
- `evaluate_suite.py` — runs `desi.diagnostics.compute_all` and
  `desi.phase_detector.detect_phases` on every fixture in a suite
  directory; writes a markdown report + machine-readable metrics JSON.
- `baseline_report.md` / `outputs/generalization_loop/baseline_metrics.json`
  — baseline (no-cycle) result against the n=20 unseen suite.
- `baseline_report_adv.md` / `outputs/generalization_loop/baseline_metrics_adv.json`
  — same baseline against the original n=10 adversarial suite, for
  cross-suite reference (3 phase overlaps were already present before
  any generalization-loop change).
- `cycle_N/proposal.md` + `cycle_N/evaluation.md` — per-cycle artefacts.
- `final_report.md` — synthesis answering the user's 12 questions.

## Reproducing the baseline

```bash
python3 experiments/generalization_loop/generate_fixtures.py
PYTHONPATH=src python3 experiments/generalization_loop/evaluate_suite.py \
  --suite data/generalization \
  --out-json outputs/generalization_loop/baseline_metrics.json \
  --out-md   experiments/generalization_loop/baseline_report.md \
  --label    baseline
PYTHONPATH=src python3 experiments/generalization_loop/evaluate_suite.py \
  --suite data/adversarial \
  --out-json outputs/generalization_loop/baseline_metrics_adv.json \
  --out-md   experiments/generalization_loop/baseline_report_adv.md \
  --label    baseline_adv_n10
```

The 20 `gen??_*.json` fixtures in `data/generalization/` are
deterministically regenerated by `generate_fixtures.py`; the
`outputs/generalization_loop/baseline_metrics*.json` files are
regenerated by `evaluate_suite.py`. The raw JSON outputs are NOT
committed (~130 KB total) — the source-of-truth is the generator + the
evaluator; the committed markdown reports capture the per-fixture
detail. To audit-by-diff, run the two commands above on a fresh
checkout and diff the regenerated artifacts byte-for-byte.

## Strict rules (per the loop brief)

- One change per cycle.
- No prompt + detector change in the same cycle.
- No silent metric changes.
- No deleting failed attempts.
- No changing fixture expectations after seeing results unless explicitly
  documented as fixture-defect correction.
- Every rejected idea remains documented.
- If questions arise, ask DESi first (self-diagnosis) — human intervention
  only for safety or repository integrity.
- Do NOT merge to `main`. All work lives on
  `experiment/desi-generalization-loop-12`.
- Optimize for honest measurement, not pretty reports.
