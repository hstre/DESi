# The Wild Brother — top critiques & proposals

He is rewarded for QUESTIONING, not for being right. Governance (the fitness harness) decided which proposals to act on. He may attack anything EXCEPT the protected core.

## Role 1 — the confused user

- **entry_point:** “I open the repo and see a 704-line paper. What is this? Where do I start?”
- **purpose:** “Why do I need this? What does it actually DO for me?”
- **first_action:** “There's no 'start here'. What's my first click?”
- **payoff:** “If I use it, what do I get back in the next 2 minutes?”

## Role 2 — the lazy user

- **jargon:** “'Epistemic state', 'drift', 'concept gate', 'recoverability' — too many words.”
- **setup:** “Too many steps. I don't want to read docs to run one thing.”
- **commands:** “I have to know Python module paths and sys.path hacks just to run a check.”
- **config:** “Too many settings. Give me one button.”

## Role 3 — the heretical user

- **real_use:** “Would anyone actually use this, or is it only for the paper?”
- **value:** “Would I pay for this? What's the concrete payoff vs a linter?”
- **governance_features:** “All these Class-A verdicts — does a user care? No.”
- **demos:** “The probes are research artifacts. Where's the thing I run on MY file?”

## Role 4 — the visionary

- **scope:** “You're selling 'governance'. Sell me a memory, a coach, an assistant.”
